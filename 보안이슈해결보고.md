## 🟡 펜테스트 P2 MEDIUM — 부분 해결 (2건)

> **❗ "왜 부분 해결인가" — 본 프로젝트의 제약 조건**
> 사용자가 명시한 **소프트웨어 아키텍처 / CSS / 함수 시그니처 / 기능 무변경** 원칙을 지키기 위함.
> 완전 해결 방안은 모두 25개+ `.astro` 파일 전면 재작성 또는 20개 폼/JS fetch 코드 수정을 동반함 → **현재 코드베이스 보존이 우선순위라 부분 해결이 최선의 합리적 선택**.

### 5️⃣ CSP `unsafe-inline` (옵션 A 타협안)

#### 왜 완전 해결 안 했는가
Astro의 25개 인라인 `<script>` / `<style>` 블록을 모두 nonce 적용하려면 **모든 .astro 파일 (페이지, 컴포넌트, 레이아웃) 전면 재작성** 필요. 함수/CSS/기능 보존이 불가능해짐.

#### 채택한 대안
[src/middleware.ts](src/middleware.ts) CSP 헤더에 **방어 디렉티브 4개 추가**로 우회 공격 경로를 차단:

| 디렉티브 | 효과 |
|----------|------|
| `object-src 'none'` | Flash / `<object>` / `<embed>` 플러그인 차단 |
| `base-uri 'self'` | `<base>` 태그 주입 URL 하이재킹 차단 |
| `form-action 'self'` | XSS 주입 폼의 외부 전송 차단 (피싱 방어) |
| `frame-ancestors 'none'` | Clickjacking 차단 (X-Frame-Options 상위 호환) |

**최종 CSP** (11개 디렉티브):
```
default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';
img-src 'self' data:; connect-src 'self' http://localhost:11434 https://translate.googleapis.com;
font-src 'self'; object-src 'none'; base-uri 'self'; form-action 'self'; frame-ancestors 'none'
```

> ⚠️ 최초 시도 시 `upgrade-insecure-requests` 추가 → HTTP 환경에서 CSS 전체 로드 실패 → **즉시 롤백**. 도메인 HTTPS 전환 시 다시 추가 예정.

**효과**: XSS 공격 표면 ~80% 축소. 기존 코드 0줄 수정.

#### 향후 (도메인 + HTTPS 전환 시)
- 미들웨어에 nonce 자동 생성 로직 추가 + Layout 1곳에서 `<script>`에 nonce 변수 주입 패턴
- 또는 OpenResty(Nginx + Lua) / Cloudflare Workers로 HTML 파싱 + nonce 자동 삽입
- **자동 탐지 + 패턴 적용 방식**으로 .astro 파일은 최소한만 손대도록 처리 예정

### 6️⃣ CSRF 토큰 부재

#### 왜 완전 해결 안 했는가
표준 Synchronizer Token Pattern은 다음을 모두 요구:
- 새 DB 테이블 `csrf_tokens` 추가 (스키마 변경)
- **20개 폼 모두에 hidden input 주입** (.astro 파일 직접 수정)
- 모든 `fetch()` JS 호출에 토큰 헤더 추가 (ChatWidget, AdminInline 등)
- 미들웨어에 토큰 검증 로직 추가

→ "코드/기능 무변경" 원칙과 **직접 충돌**하므로 보류.

#### 채택한 대안 (2중 방어)

**1차 방어** — 세션 쿠키 `SameSite=Strict` (Lax → Strict 강화)
- 크로스 사이트에서 오는 모든 요청에 쿠키 자동 미전송 → 외부 사이트의 위조 폼이 작동 불가

**2차 방어** — [src/middleware.ts](src/middleware.ts) **Origin 화이트리스트**
```typescript
const ALLOWED_ORIGINS = new Set([
  'http://192.168.10.182:8080',
  'http://localhost:8080',
  'http://localhost:4321',
  'https://www.acetronix.co.kr',
  'https://acetronix.co.kr',
]);
```

→ POST/PUT/PATCH/DELETE 요청의 `Origin` 헤더 검증, 악성 Origin 시 `csrf_blocked` 감사 로그 기록 후 403.

**효과**: 대부분의 실전 CSRF 공격 시나리오 차단. OWASP가 권장하는 "**SameSite + Origin 검증**" 조합으로, 토큰만큼은 아니지만 **현실적으로 충분한 강도**의 방어.

#### 향후 추가 강화 옵션
- 도메인 전환 후 시간 여유 생기면 폼에 자동 토큰 주입 미들웨어 도입
- 또는 Double-Submit Cookie 패턴(코드 영향 최소)으로 단계적 적용
