# AceTec Website v8

## Overview

AceTec (acetronix.co.kr) 한국 B2B 임베디드 컴퓨팅 기업의 마케팅 웹사이트.
Astro 6 기반 하이브리드(SSG + SSR). 로컬 AI 챗봇(RAG) + Admin CMS 포함.

## Tech Stack

- **Framework**: Astro 6.1.2 (server mode, prerender for static pages)
- **Language**: TypeScript (strict mode)
- **AI Chat**: Ollama local (exaone3.5:7.8b for chat, nomic-embed-text for embeddings)
- **Vector Store**: Local JSON file (data/vector-store.json, cosine similarity)
- **Admin Auth**: better-sqlite3 + bcryptjs (data/acetec.db)
- **Admin CMS**: Inline editing on pages with JSON content files (src/content/pages/)
- **Contact DB**: Optional Supabase (graceful fallback if not configured)
- **Image Processing**: sharp (product image upload, auto-crop to 4:3)
- **Styling**: Vanilla CSS with design tokens (src/styles/tokens.css)
- **Runtime**: Node.js >= 22.12.0

## Architecture

- **Static pages** (SSG, prerender=true): marketing pages, product pages, 404
- **Dynamic pages** (SSR): index (admin CMS), admin dashboard
- **API routes** (SSR): /api/chat, /api/contact, /api/health, /api/auth/_, /api/products/_, /api/pages/\*
- **RAG pipeline**: Content Collections JSON -> chunk builder -> Ollama nomic-embed-text -> local JSON vector store -> cosine similarity search -> Ollama exaone3.5:7.8b generation
- **Chat UI**: Floating pill button (bottom center) -> expandable chat panel, localStorage persistence, Korean IME handling

## Design System

- All design tokens in src/styles/tokens.css (single source of truth)
- Components use scoped `<style>` blocks
- Dynamic elements (chat messages) use `<style is:global>`
- Global utilities in src/styles/global.css
- 4 breakpoints: 1024px (tablet), 768px (mobile), 480px (small mobile)

## Conventions

- Korean comments are intentional (preserve them)
- Component Props interfaces use TypeScript
- CSS values should reference design tokens where possible
- API input validated with Zod schemas
- Admin pages protected by middleware auth check
- Ollama URL/model hardcoded in lib/ (TODO: move to env vars)
